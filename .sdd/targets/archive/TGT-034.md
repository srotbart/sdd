# Target: Gaps screen — live data fetch and archived section

## Current statement
Two changes to the Gaps screen:

1. **Live data** — `App.tsx` fetches `GET /workspaces/:id/gaps` when the active workspace changes (same trigger as the targets fetch) and passes the result to the `Gaps` component, replacing `MOCK_GAPS`. The API response fields are mapped to the frontend `Gap` type.

2. **Archived section** — the Gaps list splits gaps into active (`open` status) and archived (`closed`, `deferred`, `accepted`) sections. Active gaps appear first. Below them, a collapsible `list-divider` (same eyebrow style as the Targets archived divider — `· ARCHIVED N ·` with flanking `<hr>` rules, `letter-spacing: 0.18em`, `font-weight: 500`, collapsible with ▾/▸) separates archived gaps. Archived gap rows use `opacity: 0.55` (rising to `0.85` on hover). The domain filter applies to both sections.
