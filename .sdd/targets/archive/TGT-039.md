# Target: WorkItems ArchiveFooter — collapsible bottom strip for done/abandoned items

## Current statement
The WorkItems kanban "done · today" column shows all `done` items regardless of when they were closed, and there is no archive section. Two changes are required:

**1. Done-today split** — the "done · today" kanban column shows only items with `status: done` whose `closed` timestamp is within the last 24 hours. Items with `status: done` closed more than 24 hours ago, plus all items with `status: abandoned`, are removed from the kanban and shown exclusively in the ArchiveFooter.

**2. ArchiveFooter component** — a `hub/client/src/components/ArchiveFooter.tsx` component pinned to the bottom of the kanban area. Toggle bar styling matches `ArtifactList` eyebrow treatment: `letter-spacing: 0.18em`, `font-weight: 500`, `text-transform: uppercase`. Full spec:

- **Collapsed** (default, 40px tall): `▸ archive` caret + label + count in mono + "last closed N ago" in italic serif. Clicking anywhere on the bar expands.
- **Expanded** (max-height 320px, animated): bar shows `▾` caret; bar gains search input (placeholder "search archive…" with `⌕` glyph), status filter buttons (all / done / abandoned with counts), and domain filter buttons. Body area scrolls horizontally, grouped by day (Today / Yesterday / weekday + date label). Each day group is a vertical column of `arch-card` items.
- **arch-card** (240px wide): ID in mono accent, status pill, serif title, agent chip or "unassigned", gap ID link. `done` cards have a `var(--st-done)` left border; `abandoned` cards have `var(--ink-4)` left border at 0.85 opacity. Clicking a card opens it in the existing right-side drawer. Active card is highlighted.
- Filters reset when collapsing. Search filters across `id`, `title`, and `scope`.

`WorkItems.tsx` renders `<ArchiveFooter>` as the last child of the left column, below the kanban grid.
