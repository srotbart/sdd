# Target: Spec item files require Invariant and Acceptance Criteria sections

## Current statement

Each spec item file (`.sdd/specs/{domain}/SPEC-{abbrev}-{seq}.md`) must have three structured sections after the frontmatter:

1. `# {id} — {title}` — the heading (already present)
2. `## Invariant` — a concise statement of the rule or behavior the spec item asserts; replaces the current free-form prose body
3. `## Acceptance criteria` — a list of verifiable conditions that must hold for the invariant to be considered satisfied

All 117 existing spec item files must be migrated to this structure. Skills that write spec items (`sdd:target-engage`, `sdd:gap-to-work-items`, `sdd:work-item-close`, `sdd:spec-audit`) must produce the new structure. The Hub API spec parser must extract and expose both sections. The `references/schemas.md` file must be updated to reflect the new required sections.
