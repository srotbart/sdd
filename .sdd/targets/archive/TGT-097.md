# Target: "Improvements" artifact type produced by an improvements team

## Current statement

Introduce a new SDD artifact type, **improvements**, surfaced by a dedicated team and
followed up interactively in the main Claude session with the user. This is the
enhancement-focused sibling of the issues artifact (TGT-096).

1. **Improvements team** — a skill/command spawns a team of **3 agents via `TeamCreate`**
   whose focus is to find improvements (enhancements, refactors, simplifications,
   performance, ergonomics, better patterns) rather than defects.

2. **Improvement artifacts** — each proposal is written as an `improvement` artifact
   under `.sdd/` (e.g. `.sdd/improvements/IMP-{domain}-{seq}.md`), with enough detail
   (what to improve, where, expected benefit) to be acted on later. The three agents
   divide the work and findings are de-duplicated into distinct artifacts.

3. **Human-in-the-loop follow-up** — improvement artifacts are not auto-applied. They are
   followed up by the **main Claude session together with the user**, mirroring the
   target flow: the user and the main session **engage** an improvement (discuss/refine/
   accept/dismiss) in-document, then the outcome **mutates or creates a spec or a gap** —
   an accepted improvement that changes intended behavior becomes a spec change; one that
   identifies work needed against current spec becomes a gap; rejected ones are archived.

Relationship: shares artifact mechanics and the engage→spec/gap flow with the issues
artifact in TGT-096; differs only in intent (enhancements vs. problems). The two should
be designed together so a single mechanism backs both.

Open decisions to settle at target-engage (not pinned here):
- **Artifact schema & ID convention** — fields, status/lifecycle states, ID prefix,
  storage path and `archive/` handling (aligned with TGT-096).
- **Trigger & orchestration** — the skill/command that spawns the team; how the 3 agents
  partition the work; cross-agent dedup; autonomous vs. hub-reporting.
- **Engage flow** — the skill that drives user+session engagement and the rules routing
  an engaged improvement to a new/edited spec item vs. a gap (vs. dismissal).
- **Prioritization** — whether improvements carry an effort/impact estimate for triage.

Scope note: extends the SDD pipeline (`workflow` domain) with a new artifact type and
its team/engage lifecycle. Exact decomposition settled during target-engage.
