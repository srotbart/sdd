# Target: Spec file structure — one file per item in domain subdirectory

## Current statement

Restructure `.sdd/specs/` from one flat file per domain to one file per spec item inside a domain subdirectory. Structure: `.sdd/specs/{domain}/SPEC-{abbrev}-{NNN}.md`, with an `archive/` subdirectory per domain for removed/obsolete items. No domain manifest file — all domain metadata (`id`, `domain`, `abbrev`, `status`, `aliases`) lives in each item's own frontmatter. `version` moves from domain-level to per-item (hash of that item file's content). Gap files' `audit-spec-version` updates to reference the per-item hash; stale-audit detection in `sdd:session-start` compares against the individual item file hash. All existing flat `SPEC-*.md` domain files are deleted and replaced with migrated per-item files. All SDD skills that read or write specs are updated to walk the subdirectory structure. Hub API endpoints that return spec data are updated accordingly.
