# Target: Command palette — fuzzy search across workspace artifacts

## Current statement
The design includes a `CommandPalette` component: a full-screen overlay modal triggered by `⌘K` / `Ctrl+K` (or a search button in the toolbar). It provides fuzzy search across all artifact types in the active workspace (targets, specs, spec items, gaps, work items).

**Layout:** Overlay (`cp-overlay`) that closes on outside click. Centered dialog (`cp`) with three regions: (1) an input row with a `⌕` glyph, text input, and a scope chip showing the active workspace name; (2) a scrollable results pane; (3) a footer row with keyboard hints (↑↓ navigate, ↵ open, Esc close) and a result count.

**Default state (no query):** Shows "Recent" items — up to 4 targets, 3 gaps, 3 work items — with group headers prefixed "Recent".

**Search state:** Scores every item across ID (1.6×), title (1.2×), and full searchable text (0.7×). Exact substring scores higher than subsequence. Top 30 results grouped by kind in order: target, gap, work, spec-item, spec. Match characters are wrapped in `<mark class="cp-mark">`.

**Keyboard navigation:** ArrowUp/Down moves selection; Enter navigates to the selected item's tab and closes the palette; Escape closes. Selected row is highlighted and scrolled into view.

**Row anatomy:** kind glyph (▸ target, ≠ gap, □ work item, ∎ spec/spec-item), ID in mono accent, title, status pill, domain or location, and linked artifact ID where applicable.

**Corpus:** Built once on mount from the active workspace's targets, specs (including individual spec items), gaps, and work items. Navigation calls `onNav(navTab, navFocus)` which switches the workspace view to the correct screen and selects the item.
