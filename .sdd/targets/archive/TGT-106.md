# Target: Dedicated Hub screens for issues and improvements, each with its own artifact list

## Current statement

The Hub should have dedicated screens for the **issues** and **improvements** artifacts
(SPEC-wf-025 / SPEC-wf-026), each with its own artifact list, mirroring the existing
screen pattern used for Targets / Gaps / Work Items / Specs.

- The **issues** screen lists `ISS-*` artifacts; the **improvements** screen lists `IMP-*`
  artifacts. Each row shows the artifact's key fields and status; selecting a row shows its
  detail.
- Navigation entries are added so both screens are reachable like the other artifact screens.
- The lists reflect the artifacts' real fields — e.g. issue `severity`, improvement
  effort/impact — and their lifecycle/state.

Open decisions to settle at target-engage (not pinned here):
- **One combined screen vs. two separate screens** for issues and improvements.
- **Columns/fields** shown per row and in detail; grouping/filtering (by domain, status,
  severity).
- Whether the **engage flow** (`/sdd:review-engage`) is surfaced from the UI or the screens
  are read-only.
- Where the **nav entries** sit and how they relate to the existing pipeline screens.
- The **data layer**: how the Hub server loads `.sdd/issues/` and `.sdd/improvements/`
  (these artifact directories are new; the loader/types must be extended).

Scope note: Hub UI (`ui-screens`), depends on the issues/improvements artifacts existing
(SPEC-wf-025/026). Exact decomposition settled during target-engage.
