# Target: UI refreshes automatically when SDD artifacts change on disk

## Current statement
When SDD artifact files change on disk (targets, specs, gaps, work items — in any workspace's `.sdd/` directory), the UI must update automatically without requiring a manual page refresh.

**Server side** — the existing chokidar watcher (SPEC-arch-004) already watches the workspace `.sdd/` directory. When it detects a change to a file under `targets/`, `specs/`, `gaps/`, or `work-items/` (including their `archive/` subdirectories), the server broadcasts a WebSocket message to all connected UI clients (SPEC-arch-005) with the event type and affected workspace ID. Message shape: `{ type: "sdd-changed", workspaceId: string, artifact: "targets" | "specs" | "gaps" | "work-items" }`. Changes are debounced at the existing 150–300ms interval.

**Client side** — the UI WebSocket connection (already present for the hub dot indicator) listens for `sdd-changed` messages. When a message arrives for the active workspace, the client re-fetches the affected data endpoint (`/workspaces/:id/targets`, `/workspaces/:id/specs`, `/workspaces/:id/gaps`, or `/workspaces/:id/work-items`) and replaces the corresponding live state. Other workspaces' data is not re-fetched until they become active.
