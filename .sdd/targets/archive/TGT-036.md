# Target: Persist selected workspace in localStorage

## Current statement
The active workspace selection is stored in React state only and is lost on page refresh. The selected workspace ID must be persisted to `localStorage` under the key `hub.activeWorkspaceId`. On mount, `App.tsx` reads this key and pre-selects the matching workspace if it exists and is still present in the fetched workspace list. When the user switches workspaces, the new ID is written to `localStorage` immediately. If the persisted ID no longer matches any workspace (e.g. it was deleted), the app falls back to the first workspace in the list.

## Dialog
### 2026-05-17 — User
The selected workspace should be persisted in localStorage so it survives page refresh.
