# Target: ArtifactList archivedKey prop is unspecced — update SPEC-uic-009

## Current statement

During the round-3 implementation of SPEC-uic-009, the worker introduced an `archivedKey` prop to `ArtifactList` that is not described in the spec. The current `ArtifactListProps` interface includes:

```ts
filterKey?: keyof T;      // field to derive filter tabs from
archivedValues?: string[];// values that count as archived
archivedKey?: keyof T;    // UNSPECCED: separate field to check against archivedValues
```

`archivedKey` was introduced to support `Gaps.tsx` which filters tabs by `domain` but archives rows by `status` — two different dimensions. Without `archivedKey`, the same field must serve both as the tab grouping key and the archive classification key. SPEC-uic-009 assumes a single dimension: `archivedValues` are "values of `filterKey` that count as archived."

The spec must be updated to describe `archivedKey` as an optional override: when provided, archived rows are classified by checking `item[archivedKey]` against `archivedValues`; when absent, `filterKey` is used for both tab grouping and archive classification. The Gaps.tsx usage (`filterKey="domain"`, `archivedKey="status"`, `archivedValues={['closed','deferred','accepted']}`) is the reference implementation for this two-dimension pattern.
