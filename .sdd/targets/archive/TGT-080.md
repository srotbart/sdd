# Target: Spec item fuzzy search and dedicated detail view

## Current statement

Enhance the Specs screen (`hub/client/src/screens/Specs.tsx`) with two features scoped to the selected domain's item list:

1. **Fuzzy search** — a text input above the item list that filters spec items within the currently selected domain in real time. Implemented with `fuse.js`, searching over `id` and `title` fields only (`threshold: 0.4`). Empty query returns all items in original order. Switching domains resets the search query; pressing back from the detail view preserves it.

2. **Dedicated item detail view** — clicking a spec item replaces the item list with a full-content view. Contains: a back button (`← items`) that returns to the filtered list; the item header (ID, `StatusPill`, gap pill, `TestStatusDot`); the title; the item body rendered as markdown via `react-markdown`; and the gap/WI ref pills. Markdown is styled via a wrapper class (`.spec-item-detail__body`) for prose rhythm — no custom component overrides.

New files: `SpecItemList.tsx` + `.css`, `SpecItemDetail.tsx` + `.css`, `useSpecSearch.ts`. New dependencies: `fuse.js`, `react-markdown`. No server changes — `SpecItem.body` already contains raw markdown. `Spec` and `SpecItem` types are unchanged.

## Dialog
