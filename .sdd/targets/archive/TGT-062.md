# Target: Activity.tsx LIVE_EXTRAS fake events removed — live log stream must contain only real agent events

## Current statement

`Activity.tsx:10-16` defines a `LIVE_EXTRAS` hardcoded array of fake activity lines. `Activity.tsx:25-38` runs a `setInterval` every 6500ms incrementing `tickCount`, used to generate simulated activity entries via `liveExtras` memo. `Activity.tsx:40` then merges these fake entries with real live data: `allLines = [...liveExtras, ...lines]`.

SPEC-scr-029 requires the Activity screen's live log stream to be "populated with real agent events." Mixing hardcoded fake events with real WebSocket activity events violates this requirement and makes the log misleading.

Fix: delete `LIVE_EXTRAS`, `tickCount` state, the `setInterval` simulation, and the `liveExtras` memo. `allLines` should equal `lines` (the real `liveActivity` prop) directly. If the screen appears empty without fake data during development, that is the correct behavior — it means no agents have emitted activity events yet.
