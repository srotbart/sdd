# Target: GET /recent-workspaces server endpoint

## Current statement

`AttachWorkspaceDialog.tsx:29` calls `GET /recent-workspaces` on mount to populate the RECENT FOLDERS section (per SPEC-ui-016). No handler for this route exists in `hub/server/index.ts` — the server returns 404. The test suite mocks this endpoint so client tests pass, masking the missing implementation.

The server must implement `GET /recent-workspaces` returning the N most recently attached workspaces (by `created_at` descending, default N=5) with fields: `{ id, name, path, hasSdd: boolean }`. `hasSdd` is determined by checking whether `{path}/.sdd/` exists on disk. The response is an array (empty array if no workspaces exist). No authentication required (consistent with SPEC-arch-007).
