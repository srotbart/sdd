# Target: Spec item files support an optional subject subdirectory for intra-domain grouping

## Current statement

Within a domain directory, spec item files may optionally be grouped under a subject subdirectory: `.sdd/specs/{domain}/{subject}/SPEC-{abbrev}-{seq}.md`. The subject directory name is camelCase derived from the subject (e.g. `artifactList`, `commandPalette`). Spec items that do not belong to a specific subject remain at the domain root: `.sdd/specs/{domain}/SPEC-{abbrev}-{seq}.md`. The archive sibling lives at the same level as the spec file: `.sdd/specs/{domain}/{subject}/archive/` for subject-grouped items, `.sdd/specs/{domain}/archive/` for root-level items. The spec item ID and frontmatter are unchanged — subject is purely a filesystem organisational concern. All SDD skills that glob for spec files must walk both the domain root and any subject subdirectories (excluding `archive/`). The Hub API spec parser must find files in subject subdirectories. `sdd:session-start` stale-audit detection must resolve spec item files via both paths.
