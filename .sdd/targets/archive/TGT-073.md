# Target: Dashboard.tsx computeTotals crashes when workspace.counts is absent

## Current statement

`Dashboard.tsx:17` — `computeTotals(workspaces)` accesses `w.counts.targetsAwaitingUser` (and other counts fields) directly without optional chaining. When a workspace object has `counts: undefined` — which occurs when a WebSocket snapshot delivers a bare workspace row — the function throws `TypeError: Cannot read properties of undefined`. This crashes the Dashboard render and causes the test suite to exit with code 1 despite all 231 tests passing.

SPEC-scr-035 required optional chaining for `ws.counts` accesses in `App.tsx` — that fix was applied there, but `Dashboard.tsx` was not updated. Every `w.counts.*` access in `computeTotals` and in the JSX of `Dashboard.tsx` must use `w.counts?.field ?? 0` (for numeric fields) to be safe.

The `App.test.tsx` snapshot test fixture (`snapshotWs` at line 663) must also be updated to include a complete `counts` object matching `WorkspaceCounts`, making the test fixture representative of real server output and preventing the crash.
