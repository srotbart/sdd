# Target: sdd:explain spawns sdd-explainer via the Agent tool (no team setup)

## Current statement

The `sdd:explain` skill spawns the `sdd-explainer` agent using the Agent tool with
`name: "sdd-explainer"`, `subagent_type: "general-purpose"`, `model: "sonnet"`, and
`run_in_background: true`. It performs no team setup: it does not derive a
`sdd-explain-{project-slug}` team name, does not call `TeamCreate`, and does not pass a
`team_name` parameter to the Agent tool. As of Claude Code v2.1.178 the `TeamCreate`/
`TeamDelete` tools no longer exist; team context is created automatically when the agent is
spawned and removed automatically when the session exits, and the `team_name` input is
accepted but ignored. The agent inherits the Skill tool from its `general-purpose` agent
type. All other behavior of the skill is unchanged: it still asks interactive vs
non-interactive first, writes a skeleton to `.sdd/projections/<subject>.md` immediately,
consults `.sdd/specs/` before code, and writes sections incrementally.

This mirrors the change made to `spawn-sdd-worker` under TGT-117, applied to the
`sdd-explainer` spawn governed by SPEC-wf-020.
