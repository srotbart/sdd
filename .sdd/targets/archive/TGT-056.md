# Target: Every skill output includes a concrete next-step footer

## Current statement

Each SDD skill produces output when it completes, but none of the skill definitions require that output to include a "what to do next" footer. The user is left to remember the pipeline order themselves. This breaks the self-guiding nature of the workflow — a user who just ran `/sdd:spec-audit` should not need to know from memory that the next step is `/sdd:gap-to-work-items`.

Every skill that represents a pipeline stage must end its output with an explicit next-step suggestion in a consistent format. The suggestion must be concrete (the exact command to run, with the domain or artifact ID substituted in), conditional on the outcome (if gaps were found: X; if none: Y), and actionable without consulting documentation.

The required next-step footer per skill:

| Skill | If work found | If nothing found |
|---|---|---|
| `sdd:session-start` | Point to highest-priority action (already partially implemented via footer) | "All clear. No open targets, gaps, or work items." |
| `sdd:target-engage` | After flipping to `ready`: "Run `/sdd:target-engage {id}` to reconcile with spec." After accepting: "Run `/sdd:spec-audit {domain}` to check the codebase." | — |
| `sdd:spec-audit` | "Run `/sdd:gap-to-work-items {domain}` to decompose gaps into work items." | "No gaps found. Spec holds. Run `/sdd:spec-test {domain}` to add test coverage." |
| `sdd:gap-to-work-items` | "Run `/sdd:work-item-close {first-WI-id}` to begin implementation." | "No work items created." |
| `sdd:work-item-close` | If more open items remain: "Run `/sdd:work-item-close {next-id}` to continue." If all closed: "All work items closed. Run `/sdd:spec-audit {domain}` to verify no regressions." | — |
| `sdd:spec-test` | "Tests written. Run the test suite to verify, then run `/sdd:session-start` to check overall state." | — |
| `sdd:spec-collapse` | "Proposal written. Review `.sdd/targets/{id}.collapse.md`, then apply or discard." | — |
| `sdd:spawn-sdd-worker` | "Worker running. You will be notified on completion or blocker." | — |

The footer format must be consistent across all skills:

```
---
Next: {concrete action sentence}. Run `/{command} {arg}` to proceed.
```

Each skill's `SKILL.md` must be updated to include this footer requirement in its output section, with the exact conditional logic specified.
