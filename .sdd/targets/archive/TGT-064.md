# Target: Missing explicit background on row-level CSS classes violating SPEC-uic-006

## Current statement

SPEC-uic-006 requires every component to declare all its own visual styles with no reliance on container inheritance. The following row-level CSS rules lack an explicit `background` declaration on their base (resting) state, relying instead on whatever the parent container happens to set:

- `hub/client/src/screens/Gaps.css` — `.gaps-row {}` has no `background` (only hover and active states do)
- `hub/client/src/screens/Specs.css` — `.specs-domain-row {}` has no `background` (only hover and active states do)
- `hub/client/src/screens/WorkItems.css` — `.kcard {}` has no `background` (only `:hover` sets `var(--paper-2)`)
- `hub/client/src/screens/Activity.css` — `.act-line {}` has no `background` (only `:hover` sets one)

Each of these must add `background: var(--paper)` to their base rule, consistent with how `.target-row` is already defined (`Targets.css:122`). This ensures the component's resting background is self-declared and renders correctly regardless of container.

Note: `.artifact-list-active-row` in `ArtifactList.css` already provides `background: var(--paper)` as a default wrapper for rows inside ArtifactList, but row components rendered outside ArtifactList (e.g., Activity lines, kanban cards) must declare their own background.
