# Target: spawn-sdd-worker, gap-to-work-items, and spec-test skills missing SPEC-wf-008 footer

## Current statement

Three pipeline skills do not comply with the SPEC-wf-008 `---\nNext: {sentence}. Run \`/{command} {arg}\` to proceed.` footer format:

**spawn-sdd-worker/SKILL.md**: The output section ends with an inline text block describing what the worker will do but has no `---` divider and no `Next:` footer. Required footer: `Next: Worker running. You will be notified on completion.`

**gap-to-work-items/SKILL.md**: Output ends with a run instruction like `Run /sdd:work-item-close WI-auth-001 to start.` — no `Next:` prefix sentence and no `to proceed.` suffix. Required: `Next: Begin closing work items. Run \`/sdd:work-item-close {first-WI-id}\` to proceed.`

**spec-test/SKILL.md**: Output ends with `Run the test suite then /sdd:session-start...` — no `Next:` prefix. Required: `Next: Run the test suite to verify, then check overall state. Run \`/sdd:session-start\` to proceed.`

All three skills must be updated to add the `---` divider and `Next:` prefix sentence in their output section.
