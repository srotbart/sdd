# Target: Workspace dropdown and Attach Workspace dialog

## Current statement
The sidenav WORKSPACE section is reworked into a dropdown pattern with an attach dialog:

**Workspace dropdown (Image #3):**
The active workspace row becomes a dropdown trigger — it shows the live/idle dot, workspace name, alert badge, and a `▴` chevron. Clicking it opens a dropdown panel directly below the trigger (hairline border, paper-2 background, slight shadow). The panel lists all attached workspaces as rows: live/idle dot, workspace name (bold), path in mono below, and alert badge on the right. The active workspace row has a 2 px accent left border. A "+ attach workspace" row sits at the bottom of the panel separated by a hairline. Clicking anywhere outside the panel closes it. Clicking a workspace selects it and closes the panel.

**Attach Workspace dialog (Image #4):**
Clicking the `+` button in the WORKSPACE eyebrow, or the "+ attach workspace" row in the dropdown, opens a modal dialog. The dialog has:
- Title: *Attach workspace* in Newsreader italic (large, ~28px).
- A hairline rule below the title.
- **PROJECT LOCATION** section (uppercase eyebrow): a full-width text input with placeholder `~/code/your-project` and a `browse…` ghost link to the right (triggers a future file-picker; no-op for now).
- **RECENT FOLDERS** section (uppercase eyebrow, paper-2 background card): a list of recently seen folders, each row showing path in JetBrains Mono, a description in ink-2, and a status chip on the right — either `● FRESH` (ochre, `--st-open`) or `● SDD READY` (moss, `--st-done`) in small-caps mono. Clicking a row populates the path input.
- Footer: `Esc to cancel` hint on the left; `cancel` ghost button and a solid `attach workspace` primary button on the right.
- `×` close button in the top-right corner. Pressing Escape or clicking the backdrop closes the dialog.
- On submit, calls `POST /workspaces` with `{ name, path }` (name defaults to the last path segment).
