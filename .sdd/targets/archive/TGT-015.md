# Target: Serve parsed spec files via API

## Current statement
The server exposes `GET /workspaces/:id/specs` which reads all `SPEC-*.md` files from `{workspace.path}/.sdd/specs/`, parses their frontmatter (`id`, `domain`, `abbrev`, `version`) and each `## SPEC-{abbrev}-{seq}` item (title, status, body, ref pills), and returns a JSON array matching the `Spec[]` type the UI already expects. The Specs screen fetches from this endpoint instead of using mock data.
