# Target: sdd-worker prompt defines clear role, responsibilities, and gap reporting to lead

## Current statement
The prompt passed to the sdd-worker agent when spawned must be improved in two ways:

**1. Role and responsibilities in the system prompt** — the prompt must open with an explicit identity and responsibility declaration that cannot be confused with other agent roles. The worker must know it is a pure execution agent: it only runs `sdd:spec-audit`, `sdd:gap-to-work-items`, and `sdd:work-item-close`. It never runs `sdd:session-start`, `sdd:target-engage`, or any intent-phase skill. It never modifies specs. It never spawns other agents. If it has nothing to do (no open gaps, no pending work items, no new spec items to audit), it reports that to the lead and shuts down — it does not go looking for work.

**2. Gap reporting before decomposition** — after running `sdd:spec-audit`, the worker sends a message to the team lead listing every gap found before proceeding to `sdd:gap-to-work-items`. The lead does not need to approve (the worker continues autonomously), but the lead is kept informed of what gaps were discovered, their IDs, and their locations so the session record is accurate. This replaces the current pattern where gap discovery is only visible in the final summary.
