# Target: Targets.tsx and Gaps.tsx refactored to use ArtifactList filterKey/archivedValues API

## Current statement

SPEC-uic-009 requires that after `ArtifactList` gains the `filterKey`/`archivedValues` API, callers (specifically Targets.tsx and Gaps.tsx) are refactored to use it — passing a flat unfiltered `items` array and removing their inline filter state and active/archived split logic.

Currently neither screen uses the new API:

- `Targets.tsx:313-325` — calls `<ArtifactList items={activeTargets} archivedItems={archivedTargets} ...>` with the old split API; maintains its own `filter` state and manual `activeTargets`/`archivedTargets` split; renders its own filter bar tabs inline.

- `Gaps.tsx:56-87` — calls `<ArtifactList items={activeGaps} archivedItems={archivedGaps} ...>` with the old split API; maintains its own `filterDomain` state; renders its own domain filter bar inline.

Both screens must be refactored:
- Pass all items (unfiltered) to `ArtifactList` via the `items` prop
- Pass `filterKey="status"` (Targets) or `filterKey="domain"` (Gaps)
- Pass `archivedValues` appropriate to each domain (e.g. `['accepted','archived']` for Targets, `['closed','deferred']` for Gaps)
- Pass `filterLabels` for human-readable tab names where needed
- Remove all inline filter state, filter bar JSX, and manual active/archived split logic from both screens
