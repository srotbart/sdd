# Target: Revised shell layout — full-width header, sidenav with workspace + navigate sections

## Current statement
The application shell is restructured as follows:

**Header** spans the full viewport width at the top (above both sidenav and main content). It contains: left side — app logo/name (`sdd-hub`) and breadcrumb showing current scope (`hub / workspace / tab`); right side — connected agent count, hub address (`localhost:22351`), and current date/time.

**Sidenav** sits below the header on the left, full remaining height. It contains three sections top-to-bottom:
1. A "hub overview" link row (dot indicator, no section label).
2. A **WORKSPACE** section (uppercase eyebrow label + "+" button to attach a new workspace). One row per workspace showing a live/idle dot, workspace name, and an alert badge (e.g. "2!") for items awaiting user attention. Active workspace row has a 2px accent left border.
3. A **NAVIGATE** section (uppercase eyebrow label). One nav item per screen: session, targets, specs, gaps, work items, activity, settings — each with a count badge on the right. Active item is rendered in the accent color (oxblood) with a 2px accent left border.

**Main content** sits to the right of the sidenav and below the header, scrollable. No tab bar in the header — navigation is entirely in the sidenav.
