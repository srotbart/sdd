# Target: watcher.ts chokidar requires usePolling for reliable operation on macOS tmpdir

## Current statement

`hub/server/watcher.ts:41` creates the chokidar watcher with only `ignoreInitial: true` and `persistent: true`. On macOS, chokidar 3.6.0 uses FSEvents by default, which has significant latency on `/tmp` paths (where `os.tmpdir()` resolves in tests). This causes all 4 `watcher.test.ts` tests to fail — callbacks report count 0 because FSEvents never fires during the test's synchronous window.

Adding `usePolling: true` and `interval: 50` to the chokidar options makes the watcher reliable in both test and production environments on macOS tmpdir paths. The 50 ms polling interval is low enough to not perceptibly impact UX (the server already debounces at 150–300 ms per SPEC-arch-004) while being fast enough for tests to observe events.

Fix `watcher.ts:41-45`: add `usePolling: true, interval: 50` to the chokidar options object.
