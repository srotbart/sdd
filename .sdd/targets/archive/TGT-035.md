# Target: WorkItems screen — live data fetch, done-today split, and ArchiveFooter

## Current statement
Three changes to the WorkItems screen:

1. **Live data** — `App.tsx` fetches `GET /workspaces/:id/work-items` when the active workspace changes and passes the result to the `WorkItems` component, replacing `MOCK_WORK_ITEMS`.

2. **Done-today split** — the "done · today" kanban column shows only items closed within the last 24 hours. Items with `status: done` that were closed more than 24 hours ago, plus items with `status: abandoned`, move to the ArchiveFooter and are not shown in any kanban column.

3. **ArchiveFooter** — a collapsible horizontal strip pinned to the bottom of the kanban area (above the drawer, below the columns). Collapsed state: shows `▸ archive`, item count, and "last closed N ago". Expanded state: reveals a scrollable list grouped by day (Today / Yesterday / Mon 12 May…) with search input, status filter (all / done / abandoned), and domain filter. Each archived card shows ID, status pill, title, agent chip or "unassigned", gap ID link. Clicking a card opens it in the existing right-side drawer. Filters reset when collapsing.

## Dialog
### 2026-05-17 — User
Wire WorkItems to live data, split the done column to show only items closed today, and add the ArchiveFooter at the bottom — the collapsible archive strip with search, filters, and day grouping from the design.
