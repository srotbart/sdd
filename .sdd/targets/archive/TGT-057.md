# Target: SDD plugin statusline — installable via skill to local scope

## Current statement

The SDD plugin must ship a statusline configuration installable via a dedicated skill that writes to `.claude/settings.json` (local/project scope, not `~/.claude/settings.json` user scope). The skill is invokable as `/sdd:install-statusline` from any Hub-connected Claude Code session in the project.

### Statusline layout

The statusline has two segments, separated by `|` dividers:

**Left segment — shell context** (mirrors the reference design from the screenshot):

```
→  {current-dir}  |  git:({branch}) {dirty}  |  [{context-bar}] {context-pct}%
```

- `→` — arrow indicator, color: `#5fffaf` (bright green)
- `{current-dir}` — basename of `$PWD`, color: `#5fd7ff` (cyan)
- `git:` — literal, color: `#ffd700` (yellow)
- `({branch})` — git branch name, color: `#ff87ff` (pink/magenta)
- `{dirty}` — `x` when working tree is dirty, empty when clean, color: `#ff5f5f` (red)
- Context bar `[░░░░░]` and percentage — Claude Code's built-in context usage token, color: `#808080` (gray)

**Right segment — SDD pipeline state**:

```
specs: {n}  |  targets: {awaiting}  |  gaps: {open}  |  work: {pending}/{in-progress}
```

- `specs:` label in muted gray (`#626262`), count in white
- `targets:` — count of targets with `status: awaiting-user` for the active workspace; color: `#ffd700` (yellow) when > 0, gray when 0
- `gaps:` — count of open gaps; color: `#ff5f5f` (red) when > 0, gray when 0
- `work:` — `{pending}/{in-progress}` work items; color: `#5fd7ff` (cyan) when any active, gray when both 0

SDD counts are read by shelling out to `jq` against the Hub API (`GET localhost:22351/workspaces/{id}/targets` etc.) or by counting `.sdd/` files directly if the Hub is not running. If the Hub is unreachable and no `.sdd/` directory is found, the SDD segment is omitted entirely.

### Install skill

The skill `/sdd:install-statusline` (added to `plugin/skills/install-statusline/SKILL.md`) writes the statusline configuration to `.claude/settings.json` in the current working directory. It must:

1. Read `.claude/settings.json` if it exists (create if not).
2. Merge the `statusline` key with the new configuration — do not overwrite unrelated settings.
3. Confirm the write with: `Statusline installed to .claude/settings.json (local scope). Restart Claude Code to apply.`

The skill must NOT write to `~/.claude/settings.json`. Local scope ensures the statusline is per-project and does not affect other workspaces.
