# Target: Coding standards have one home (.sdd/standards/), surfaced via session-start

## Current statement

Coding standards have a single authoritative home: the structured `.sdd/standards/`
rubric. They are **not** duplicated into `CLAUDE.md` — duplicating the rule content into a
second file invites drift between the two copies. Instead, the agent receives the standards
proactively because `sdd:session-start` surfaces the `.sdd/standards/` rubric (or a concise
reference to it) into the agent's context at session open — consistent with how session-start
already delivers the artifact operating contract (SPEC-wf-024) and high-level orientation
(SPEC-wf-030). `CLAUDE.md` may keep a brief pointer to `.sdd/standards/` so a reader knows
where the rubric lives, but it does not carry the rule content.

This supersedes SPEC-wf-028's requirement that the standards live in *both* `CLAUDE.md` and
`.sdd/standards/`, and changes SPEC-wf-029's proactive layer (layer 1) from "standards in
`CLAUDE.md` reach Claude's context" to "session-start surfaces the `.sdd/standards/` rubric
into the agent's context." The other two enforcement layers (active-blocking lint, review-time
reviewer) and the "standards are user-authored, not spec-audited" invariant are unchanged.
