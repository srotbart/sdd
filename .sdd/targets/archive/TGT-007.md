# Target: Persist workspace settings to SQLite via REST API

## Current statement
Workspace settings (name, path, description) entered in the Settings screen are persisted to the SQLite database. The server exposes two endpoints:
- `GET /workspaces` — returns all workspaces as JSON.
- `PATCH /workspaces/:id` — accepts a partial `{ name?, path?, description? }` body and updates the matching row in SQLite.

The Settings screen calls `PATCH /workspaces/:id` on each field blur. On initial load, workspace data is fetched from `GET /workspaces` and used to populate the UI. No explicit save button is required.
