# Target: Design phase — optional pre-target planning artifact in .sdd/design/

## Current statement

An optional design phase precedes target creation for larger or more complex changes. A design lives at `.sdd/design/<name>/design.md` with optional sibling files in the same directory (diagrams, research notes, alternatives). The design is free-form markdown — no fixed schema beyond a problem/solution structure. When planning converges, targets are created from the design; the design directory is never deleted. `sdd:session-start` surfaces open designs (those with no targets spawned yet) under a **Designs in progress** section. Targets may optionally carry a `design:` frontmatter reference back to the doc that spawned them.

Call it a **Feature** (`.sdd/features/FT-001.md`). Same mental model as a Target — you negotiate a statement, it gets accepted — but with more room inside: free-form sections, Mermaid diagrams, open questions, and a `spawns:` list of Target IDs it produces over time. Proposed Current statement:
