# Target: spec-audit and work-item-close skills must use the exact SPEC-wf-008 footer format

## Current statement

Both `plugin/skills/spec-audit/SKILL.md` and `plugin/skills/work-item-close/SKILL.md` produce footers that do not match the format mandated by SPEC-wf-008.

Current format (both skills): `Run \`/sdd:{command} {arg}\` to {action}.` — a bare run instruction with no preceding sentence.

Required format per SPEC-wf-008: `Next: {sentence}. Run \`/{command} {arg}\` to proceed.`

Both skills must be updated so their output section ends with a `---` divider followed by the exact format. The `Next:` prefix sentence should describe what will happen (e.g., "Next: decompose gaps into work items." or "Next: close the next work item."), and the run instruction must use `to proceed.` as the closing phrase.

Specific updates needed:
- `spec-audit`: gaps found → `Next: Decompose gaps into work items. Run \`/sdd:gap-to-work-items {domain}\` to proceed.`; no gaps → `Next: Add test coverage to spec items. Run \`/sdd:spec-test {domain}\` to proceed.`
- `work-item-close`: items remain → `Next: Continue with the next work item. Run \`/sdd:work-item-close {next-id}\` to proceed.`; all closed → `Next: Verify the spec holds. Run \`/sdd:spec-audit {domain}\` to proceed.`
