# Target: ArtifactList filter bar active tab treatment must match Targets screen style

## Current statement

The ArtifactList filter bar's active tab uses `background: var(--accent); color: var(--paper)` — a solid filled style (white text on solid accent). The Targets screen filter bar uses `color: var(--accent); background: color-mix(in srgb, var(--accent) 8%, transparent)` — a tinted-tab style (accent text on lightly tinted background).

SPEC-uic-009 requires the filter bar visual treatment to "match the Targets screen filter bar exactly." The current ArtifactList implementation does not match — it uses the wrong active state style.

Additionally, the count chip treatment differs:
- ArtifactList: `font-family: var(--mono); font-size: 10px; color: inherit; opacity: 0.7`
- Targets: no mono font, `color: var(--ink-4)`, no opacity

Fix `ArtifactList.css`:
- `.artifact-list-filter-btn--active`: change to `color: var(--accent); background: color-mix(in srgb, var(--accent) 8%, transparent);` (remove the solid fill)
- `.artifact-list-filter-count`: remove `font-family: var(--mono)` and `opacity: 0.7`; use `color: var(--ink-4)`
