# Target: spawn-sdd-worker uses the Agent tool directly (no team setup)

## Current statement

The `spawn-sdd-worker` skill spawns the `sdd-worker` agent using the Agent tool with
`name: "sdd-worker"`, `subagent_type: "general-purpose"`, `model: "sonnet"`, and
`run_in_background: true`. It performs no team setup or teardown: it does not call
`TeamCreate` or `TeamDelete` (both removed in Claude Code v2.1.178), does not derive a team
name from the project root, and does not pass a `team_name` parameter to the Agent tool (the
parameter is accepted but ignored). The team context is created automatically when the
worker is spawned and removed automatically when the session exits. The worker inherits the
Skill tool from its `general-purpose` agent type, so no setup step is required to grant Skill
access.

Scope is limited to the `sdd-worker` spawn in `spawn-sdd-worker`. The `review-issues` and
`review-improvements` skills also call the removed `TeamCreate` tool (SPEC-wf-025/026); those
are out of scope here and tracked separately.
