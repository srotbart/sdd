# Target: Sidenav alert badge wired, empty-workspace state specced, header live clock, and nav conditional rendering specced

## Current statement

Four related ui-layout gaps:

**1. Sidenav alert badge is never computed.**
`Sidenav.tsx:9` accepts `alertCount?: number` and renders it (`Sidenav.tsx:98-100`). `App.tsx:445-449` maps workspaces as `{ id, name, path }` — `alertCount` is never included. The badge never appears. SPEC-ui-002 says "alert badge for items awaiting user attention" but gives no computational definition. The spec must define that `alertCount` equals the count of active targets with `status === 'awaiting-user'` for that workspace, and `App.tsx` must populate it from `liveTargets`.

**2. Empty-workspace state is unspecced.**
When no workspaces are attached, `Sidenav.tsx:96` renders `'select workspace'` in the dropdown trigger. SPEC-ui-002 and SPEC-ui-007 both assume at least one workspace exists. No spec item describes the sidenav state when `workspaces.length === 0` — what the WORKSPACE section shows, whether the NAVIGATE section hides, and whether the main content shows a "no workspace attached" prompt.

**3. Conditional hiding of the NAVIGATE section is unspecced.**
`Sidenav.tsx:133`: `{activeWorkspaceId && <div>...NAV_TABS...</div>}` — the entire NAVIGATE section is hidden when no workspace is selected. SPEC-ui-002 describes three sidenav sections as if they are always present. The spec must explicitly state that the NAVIGATE section only renders when a workspace is active.

**4. Header clock is frozen at mount time.**
`Header.tsx:10-12` computes `now = new Date()` once at render — no `setInterval`, no `useEffect`. After an hour, the header still shows the time from page load. SPEC-ui-003 says "current date/time", implying a live clock. The spec must require the clock to update every minute via `setInterval`, and must specify the timezone (currently hard-coded as UTC via `.toISOString().slice(11,16) + ' UTC'`).
