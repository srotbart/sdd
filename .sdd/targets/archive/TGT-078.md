# Target: Gap parser derives domain from spec-item when domain field is absent; ArtifactList skips empty-string tabs

## Current statement

`sdd-parser.ts` defaults `domain` to `""` when the gap frontmatter has no `domain` field: `domain: meta["domain"] ?? ""`. Gaps written before the `domain` field was standardised (e.g. GAP-scr-019) have no `domain` key in their frontmatter, so they are parsed with `domain: ""`.

When `ArtifactList` uses `filterKey="domain"`, it derives filter tabs from unique `domain` values across all gap items. A domain of `""` creates a tab with an empty string key. The tab renders as `{filterLabels?.[''] ?? ''}` ‚Äî an empty string ‚Äî so only the count number is visible and the label is blank.

Two fixes required:

**1. `sdd-parser.ts` ‚Äî derive domain from `spec-item` when `domain` is absent.**
The `spec-item` ID encodes the domain abbreviation (e.g. `SPEC-scr-023` ‚Üí abbrev `scr`). The parser should map the abbrev back to the full domain name using the known abbrev‚Üídomain table: `arch ‚Üí architecture`, `scr ‚Üí ui-screens`, `ui ‚Üí ui-layout`, `uic ‚Üí ui-components`, `wf ‚Üí workflow`. If `meta["domain"]` is absent or empty and `meta["spec-item"]` is present, extract the abbrev from the spec-item ID and resolve the domain. If resolution fails, fall back to the abbrev itself rather than `""`.

**2. `ArtifactList.tsx` ‚Äî skip tabs where `tab === ""`.**
In the tab derivation step, filter out any empty-string values from `allValues` before building the `tabs` array. A blank tab with only a count is confusing and serves no filtering purpose.
