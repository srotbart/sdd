# Target: review skills spawn their 3 agents via the Agent tool (no TeamCreate)

## Current statement

The `sdd:review-issues` and `sdd:review-improvements` skills each spawn exactly 3 reviewer
agents using the Agent tool, not via `TeamCreate`. As of Claude Code v2.1.178 the
`TeamCreate`/`TeamDelete` tools no longer exist. The three reviewers work independently on
disjoint review lenses and report their findings back; the invoking skill collects and
de-duplicates the findings and writes the artifacts (`ISS-{domain}-{seq}` / `IMP-{domain}-{seq}`).
Because the reviewers do not need to message each other — they fan out and report back — they
are spawned as Agent-tool subagents. All other behavior is unchanged: exactly 3 agents,
disjoint lenses, de-duplication before writing, never auto-fix/auto-apply, same artifact
schemas and storage.

This mirrors TGT-117 / TGT-118, applied to the reviewer teams governed by SPEC-wf-025
(issues) and SPEC-wf-026 (improvements).
