# Target: Skill footer format deviation — spec-audit and work-item-close do not use the SPEC-wf-008 format

## Current statement

SPEC-wf-008 mandates a specific footer format for every pipeline skill:

```
---
Next: {sentence}. Run `/{command} {arg}` to proceed.
```

Both the `spec-audit` and `work-item-close` skills produce footers that omit the required `Next:` prefix and `to proceed.` suffix:

- `plugin/skills/spec-audit/SKILL.md:134-136` — ends with `Run \`/sdd:gap-to-work-items\` to decompose into work items.` (no `Next:` prefix, no `to proceed.` suffix)
- `plugin/skills/work-item-close/SKILL.md:103-108` — ends with `Run \`/sdd:session-start\` to see updated state.` (same deviation)

Both skills must be updated to use the exact format: `Next: {sentence}. Run \`/{command} {arg}\` to proceed.` — with the exact wording conditional on whether work was found or not, as specified in SPEC-wf-008.
