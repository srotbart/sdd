### 2026-06-01 — User

Implement the sdd:explain skill from the design at .sdd/design/sdd-explain/design.md. The skill spawns a dedicated sdd-explainer agent (general-purpose, sonnet) on team sdd-explain-{project-slug}. On spawn the agent asks the user: interactive or non-interactive? Then immediately writes a skeleton header to .sdd/projections/<subject>.md. In interactive mode the agent writes the main concept section first (specs → code), then asks the user what to explore next each turn, writing each section as directed. In non-interactive mode the agent traverses autonomously (specs → entry points → key components) and writes the full document without asking. Specs are always checked first as the authoritative ground truth.
