# Target: AttachWorkspaceDialog spec alignment — name field, description, SDD preview, and live recent folders

## Current statement

SPEC-ui-008 describes the AttachWorkspaceDialog but the implementation has diverged significantly. The spec needs to be updated to match the actual dialog, and two implementation bugs need fixing:

**Fields in the implementation not in the spec:**
- **Workspace name field** (`AttachWorkspaceDialog.tsx:163-174`): auto-derived from path basename, editable by user. Not mentioned in SPEC-ui-008.
- **Description field** (`AttachWorkspaceDialog.tsx:176-189`): optional text input. Not mentioned in SPEC-ui-008. Also a bug: the description is collected but never sent in the POST body (`AttachWorkspaceDialog.tsx:75-79` only sends `{ name, path }`).
- **SDD detection preview panel** (`AttachWorkspaceDialog.tsx:129-161`): shows "Existing .sdd/ detected" with artifact counts or "No .sdd/ here yet" prompt. Not mentioned in SPEC-ui-008.
- **Command peek block** (`AttachWorkspaceDialog.tsx:192-200`): shows `/sdd:init` command. Not mentioned in SPEC-ui-008.
- **"initialize & attach" primary button label variant** (`AttachWorkspaceDialog.tsx:53`): button label changes to "initialize & attach" when no `.sdd/` found. Spec says only "attach workspace".

**Implementation bugs:**
- **RECENT_FOLDERS is hardcoded mock data** (`AttachWorkspaceDialog.tsx:12-17`): three hard-coded folder entries. The spec implies dynamic data. The dialog must call an API (e.g., the existing `GET /browse-folder` or a new `GET /recent-workspaces`) to populate this.
- **Description not sent in POST body**: the description field is collected but discarded before submit. If the spec includes a description field, it must be sent.

**Action required:** Update SPEC-ui-008 to document the actual dialog fields (name, description, SDD preview panel, command peek, button label variants); fix the POST body to include `description`; and replace the hardcoded RECENT_FOLDERS with real data or remove the field from the spec.
