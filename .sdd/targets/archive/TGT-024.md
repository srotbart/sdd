# Target: spawn-sdd-worker skill should use TeamCreate to spawn worker agents

## Current statement
The `sdd:spawn-sdd-worker` skill spawns the worker via bare `Agent tool` (no team context). Background agents spawned this way do not have access to the Skill tool, which means the worker cannot invoke `sdd:spec-audit`, `sdd:gap-to-work-items`, or `sdd:work-item-close`. The skill must be updated to use `TeamCreate` + `Agent` with a `team_name` parameter so that the worker agent runs in a team context with full tool access including the Skill tool.
