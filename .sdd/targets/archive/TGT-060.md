# Target: ArtifactStatus missing 'deferred' and 'abandoned' — dead filtering logic

## Current statement

`ArtifactStatus` in `types.ts:1-15` does not include `'deferred'` or `'abandoned'`. Both values are used in filtering logic:

- `App.tsx:482-484` — gap count filtering excludes `'deferred'` status (TypeScript flags this as a dead comparison — the value never overlaps the union, so the filter never fires for deferred gaps)
- Work items use `'abandoned'` status (referenced in ArchiveFooter, WorkItems components)

TypeScript correctly reports these as dead-code comparisons because the string literals are outside the union. The filtering is silently broken: deferred gaps are counted as open, abandoned work items may be miscounted.

Both values must be added to `ArtifactStatus`: `'deferred'` for gaps, `'abandoned'` for work items. The spec pipeline (gap states: open → closed | deferred; work item states: pending → in-progress → done | abandoned) requires both terminal variant states to be representable.
