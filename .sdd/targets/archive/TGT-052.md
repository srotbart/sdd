# Target: AgentChip and StatusPill spec coverage

## Current statement

Two shared components in `hub/client/src/components/` have zero spec coverage in SPEC-uic or anywhere else:

**AgentChip** (`AgentChip.tsx`, `AgentChip.css`): renders an agent identifier pill used across WorkItems, ArchiveFooter, Session, Activity, and Dashboard. Used in at least 5 places. No spec item describes its props, visual treatment, or behavior (e.g., "unassigned" fallback, tooltip, click behavior if any).

**StatusPill** (`StatusPill.tsx`, `StatusPill.css`): renders a colored semantic status label. `STATUS_MAP` in `StatusPill.tsx:9-23` defines 12 distinct status → label/color mappings covering the full `ArtifactStatus` union (`awaiting-user`, `awaiting-agent`, `ready`, `draft`, `accepted`, `open`, `pending`, `in-progress`, `blocked`, `done`, `closed`, `active`, `stale`). This mapping is a key piece of the system's visual language with no spec coverage. No spec item defines what statuses map to what labels/colors, or that `StatusPill` is the single source of truth for status rendering.

Both components meet the SPEC-uic threshold: "shared, reusable React components consumed by two or more screens" (SPEC-uic intro, line 11).

A spec item for each is needed covering: file path, props interface, visual behavior, and the STATUS_MAP contract for StatusPill.
