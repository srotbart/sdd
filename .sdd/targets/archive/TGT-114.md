# Target: Syntax highlighting for code blocks in rendered markdown

## Current statement

The shared `Markdown` component (SPEC-uic-013) renders fenced code blocks with **syntax
highlighting** — language-aware token coloring based on the code fence's language tag —
instead of flat monospaced text, integrated into the existing react-markdown pipeline and
the app's light/dark theming.

1. Fenced code blocks with a language tag (e.g. ```` ```ts ````) render with highlighted tokens.
2. Plain fences with no language degrade gracefully — rendered as unhighlighted code, no errors.
3. An unknown / unsupported language tag degrades gracefully (unhighlighted, no errors thrown).
4. Highlight colors come from the existing `--hl-*` CSS tokens (keyword / function / string /
   number), honoring both light and dark mode — not a clashing third-party color scheme.
5. Artifact IDs inside code blocks remain NON-linkified (SPEC-uic-014 preserved), and raw HTML
   stays disabled (no `rehype-raw`) — highlighting must not require it.

Resolved decisions (see Dialog 2026-06-05 — Agent):
- **Highlighter:** `rehype-highlight` (highlight.js) — a rehype plugin that composes into the
  existing react-markdown pipeline alongside `remark-gfm` and the artifact-link remark plugin,
  emits stable `hljs-*` token classes, and needs no raw HTML.
- **Theme integration:** CSS maps `hljs-*` token classes onto the existing
  `--hl-kw` / `--hl-fn` / `--hl-str` / `--hl-num` tokens; unmapped tokens inherit the default
  code `--ink` color.
- **Language coverage:** highlight.js `common` language set, no lazy grammar loading
  (broad / lazy-loaded grammars deferred).

Scope: extends the shared `Markdown` component (`ui-components`, builds on SPEC-uic-013/014).

### 2026-06-05 — Agent
Target intent is clear; resolving the flagged open decisions inline rather than deferring.

- **Highlighter → `rehype-highlight` (highlight.js).** Drops into the existing `react-markdown`
  pipeline as a rehype plugin (no `code` renderer rewrite), emits stable `hljs-*` token classes,
  and does not require `rehype-raw` so SPEC-uic-013's raw-HTML-off safety stays intact. Shiki
  (async, ships its own TextMate theme JSON) and `react-syntax-highlighter` (own theme objects,
  replaces the `code` renderer) both fight the existing CSS-token theming and add weight for no
  gain here.
- **Theme integration → existing `--hl-*` tokens.** `src/styles/tokens.css` already defines
  `--hl-kw` / `--hl-fn` / `--hl-str` / `--hl-num` for light and dark; map the `hljs-*` token
  classes (`keyword`, `title.function`, `string`, `number`, and siblings) onto them in CSS.
  Unmapped token classes inherit `--ink`, so coloring is partial-but-coherent, never clashing.
- **Language coverage → highlight.js `common` set, no lazy loading.** Covers ts/js/json/bash/
  python/css/md etc. that appear in SDD artifacts. Broad/lazy grammar loading deferred.
- **Degradation →** configure `detect: false` + `ignoreMissing: true` so unlabeled or
  unknown-language fences render as plain unhighlighted code with no thrown errors.
