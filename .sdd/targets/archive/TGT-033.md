# Target: GET /workspaces/:id/work-items API endpoint

## Current statement
The server exposes `GET /workspaces/:id/work-items` which reads all work item files (active and archived) from `.sdd/work-items/` and `.sdd/work-items/archive/` within the workspace's `.sdd` directory and returns them as a JSON array. Each work item is parsed from its frontmatter and body. Fields returned: `id`, `gapId` (from `gap-id`), `domain`, `status`, `created`, `abandonedReason` (from `abandoned-reason`), `title` (first `# Work Item:` heading), `scope` (from `**Scope:**` body field), `acceptance` (array, from `**Acceptance criteria:**` bullet list), `progressNote` (from `**Progress:**` body field if present), `blockedReason` (from `**Blocked:**` body field if present), `closed` (from frontmatter `closed` field if present).
