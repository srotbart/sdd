# Target: spawn-sdd-worker derives unique team name from project root to prevent cross-session collisions

## Current statement
The `sdd:spawn-sdd-worker` skill always creates a team named `sdd-execution`. When two Claude sessions in different projects run the skill concurrently, they create teams with identical names and identical worker names (`sdd-worker`). The Hub's messaging system cannot distinguish between them, so completion messages from one session's worker are delivered to the other session's lead.

The skill must derive the team name from the last path segment of the project root: `sdd-{project-slug}` where `{project-slug}` is the final component of the project's root directory path (e.g. `sdd-sdd-repo` for `/Users/x/sdd-repo`, `sdd-argus-muse` for `/Users/x/argus/argus-muse`). This makes team names unique per project and eliminates cross-session message delivery collisions. The worker agent name remains `sdd-worker` but is scoped within the unique team.
