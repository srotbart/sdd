# Target: Native folder picker for Attach Workspace dialog

## Current statement
The "browse…" button in the Attach Workspace dialog triggers a native macOS folder picker via a server-side endpoint. The server exposes `GET /browse-folder` which invokes `osascript` to open the system folder-chooser dialog and returns the selected path as `{ path: string }` JSON. If the user cancels, it returns `{ path: null }`. The client calls this endpoint when "browse…" is clicked, then populates the path input with the result. The endpoint is macOS-only for v1.

## Dialog
### 2026-05-15 — User
Add ability to select a folder (browser) on attach workspace dialog.

### 2026-05-15 — Agent
Browser-based folder pickers cannot return full filesystem paths due to browser security restrictions. For this localhost-only macOS tool the correct approach is a server endpoint that calls osascript to open the native folder chooser and returns the selected path. Proposing GET /browse-folder → runs osascript → returns { path } JSON.
