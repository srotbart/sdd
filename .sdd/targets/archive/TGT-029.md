## Current statement
The design includes a Plugin Reference screen (`PluginReference` component) that lives as a permanent bottom-of-sidenav navigation item. It is a static read-only documentation page for the SDD workflow. The screen does not currently exist in the implementation.

**Screen layout:** Two-pane with a 220px left TOC sidebar and a scrollable right content pane. The toolbar shows `❡ plugin reference — SDD workflow, artifacts, and skills` with a `view source on github ↗` ghost button.

**TOC sections (7):** Overview, Pipeline, Artifacts, Status lifecycles, Skills, Schemas, Design decisions. The left sidebar has a scroll-spy that highlights the active section. Clicking a section smoothly scrolls the right pane to it.

**Sidenav placement:** A fixed entry at the bottom of the sidenav, visually separated from workspace-scoped navigation, always visible regardless of active workspace.

**Content:** All seven sections as rendered in the design's `plugin-reference.jsx`, including the pipeline ASCII diagram, artifact cards (docs-card), skills table (skill-card + skill-cmd), schema pre blocks, lifecycle tables, and design decisions list.
