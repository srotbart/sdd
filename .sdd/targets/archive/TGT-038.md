# Target: Reusable ArtifactList component with collapsible archived divider

## Current statement
The collapsible archived-section divider (eyebrow label `· ARCHIVED N ·` flanked by `<hr>` rules, `letter-spacing: 0.18em`, `font-weight: 500`, ▾/▸ toggle, `opacity: 0.55` on archived rows) is implemented inline in `Targets.tsx`. The Gaps screen needs the same treatment. Rather than duplicating the pattern, a shared `ArtifactList` component is extracted and reused.

**Component:** `hub/client/src/components/ArtifactList.tsx`

```ts
interface ArtifactListProps<T> {
  items: T[];                          // active (non-archived) items
  archivedItems: T[];                  // archived items
  renderRow: (item: T, archived: boolean) => React.ReactNode;
  getKey: (item: T) => string;
}
```

`ArtifactList` renders:
1. All `items` via `renderRow(item, false)`
2. When `archivedItems.length > 0`: the collapsible eyebrow divider (sourced verbatim from the current Targets implementation — same CSS classes, same toggle logic, same ▾/▸ caret, count in mono span)
3. When expanded: all `archivedItems` via `renderRow(item, true)` wrapped in a container that applies `opacity: 0.55` (0.85 on hover, 1 when selected — callers signal selection via a CSS class on the row element)

**Refactoring scope:**
- `Targets.tsx` replaces its inline divider + archived-row block with `<ArtifactList>`, passing active and archived targets and the existing `TargetListRow` render function. Behaviour and appearance must be identical after refactor.
- `Gaps.tsx` replaces its current plain list (no archived section) with `<ArtifactList>`, passing active (`open`) and archived (`closed`, `deferred`, `accepted`) gaps. The archived rows get the same opacity treatment.
- No other screens are in scope for this change.

**CSS:** The divider and archived-row opacity styles move to `hub/client/src/components/ArtifactList.css` and are removed from `Targets.css`. `Gaps.css` removes any duplicate archived-row opacity rules if present.
