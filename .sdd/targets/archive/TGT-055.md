# Target: Each UI component owns all its own styles — parent containers must not determine component appearance

## Current statement

Every UI component is responsible for declaring all of its own visual styles — background, color, typography, spacing, border, opacity — in its own CSS file. A component must render correctly regardless of where it is placed. No component may rely on inherited or ambient styles set by a parent container to achieve its intended appearance.

The failure mode that prompted this target: `Gaps.tsx` rows showed a different background than `Targets.tsx` rows despite both using `ArtifactList`. `.gaps-row` had no explicit `background`, so it inherited `var(--paper-2)` from `.gaps-list`. Moving the component to a different container would silently change its appearance. The same class of bug applies to any CSS property — color, font-size, opacity, border — if the component omits it and a container happens to set it.

The principle to encode across all specs:

1. **Components own all their visual properties.** Background, foreground color, typography, borders, padding, and opacity must all be declared in the component's own CSS. A component's appearance is determined by its own stylesheet, not by its ancestors.

2. **Inherited CSS properties must be explicitly reset or re-declared.** Properties that CSS inherits by default (color, font-size, font-family, line-height, etc.) must be explicitly set on each component root that has a defined visual identity. Relying on a value cascading down from a layout ancestor is not permitted.

3. **Container styles are structural, not cosmetic.** Containers (scroll areas, list panes, layout regions) may set background or color for their own structural purpose (e.g., a sidebar tint, a panel surface), but components rendered inside them must not depend on those values for their own appearance.

4. **Overrides flow from container to component, never the reverse.** When a specific placement requires a style deviation (e.g., a row inside a dark panel needs a different background), the container overrides the component's class in a scoped selector. The component's own CSS always provides a complete, self-sufficient baseline.

An audit of all component CSS files in `hub/client/src/components/` and all row elements in `hub/client/src/screens/` should verify compliance against each of the four rules above.
