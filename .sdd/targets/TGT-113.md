# Target: Sidenav shows active-artifact counts next to every artifact item

## Current statement

The sidenav shows the number of active (non-archived) artifacts next to each artifact-bearing
nav item. The count mechanism already exists (`tabCounts` → `sidenav-nav-count`, governed by
SPEC-scr-018, "counts reflect non-archived items only"), but today it only covers **targets,
gaps, work items, and specs**. Extend it so every artifact item shows its count — adding
**issues** (`ISS-*`), **improvements** (`IMP-*`), and the document artifacts **projections**,
**designs**, and **standards**.

- Counting follows the existing non-archived rule (SPEC-scr-018): e.g. targets exclude
  accepted/archived, gaps exclude closed/deferred, work items exclude done/abandoned.
- Issues/improvements counts reflect active (not-yet-addressed/archived) entries.
- Projections/designs/standards counts reflect the documents present.

Open decisions to settle at target-engage (not pinned here):
- Exact "active" semantics per type — especially issues/improvements (active = open/unaddressed)
  and the document types (count of files vs. something finer).
- Whether non-artifact tabs (session, activity, settings) intentionally stay countless.
- Extend SPEC-scr-018 vs. add a new spec item for the additional types.

Scope note: extends the existing sidenav counting (`ui-screens`, SPEC-scr-018); driven by the
`tabCounts` prop assembled in `App.tsx`. Exact decomposition settled during target-engage.
