# Target: "Issues" artifact type produced by a reviewer team

## Current statement

Introduce a new SDD artifact type, **issues**, surfaced by a dedicated reviewer team and
followed up interactively in the main Claude session with the user.

1. **Reviewer team** — a skill/command spawns a team of **3 agents via `TeamCreate`**
   whose sole purpose is to go over everything and flag potential issues. Scope of what
   counts as an issue is broad: code bugs/correctness, code styling, anti-patterns, code
   smells, bad practices, and issues with the specs themselves.

2. **Issue artifacts** — each finding is written as an `issue` artifact under `.sdd/`
   (e.g. `.sdd/issues/ISS-{domain}-{seq}.md`), with enough detail (location, what's
   wrong, why) to be acted on later. The three reviewers divide the work and their
   findings are de-duplicated into distinct artifacts.

3. **Human-in-the-loop follow-up** — issue artifacts are not auto-fixed. They are
   followed up by the **main Claude session together with the user**, mirroring the
   target flow: the user and the main session **engage** an issue (discuss/refine/accept/
   dismiss) in-document, then the outcome **mutates or creates a spec or a gap** — a
   genuine spec divergence becomes a gap; a missing/incorrect spec requirement becomes a
   spec change; non-actionable items are dismissed/archived.

Relationship: this is the defect/divergence-flagging counterpart to the improvements
artifact in TGT-097; the two should share artifact mechanics and the engage→spec/gap
flow, differing only in intent (problems vs. enhancements).

Open decisions to settle at target-engage (not pinned here):
- **Artifact schema & ID convention** — fields, status/lifecycle states, ID prefix,
  storage path and `archive/` handling.
- **Trigger & orchestration** — the skill/command that spawns the team; how the 3 agents
  partition the codebase/specs (by domain, by file area, by concern); cross-agent dedup;
  whether reviewers run autonomously or report back through the hub.
- **Engage flow** — the skill that drives user+session engagement and the rules routing
  an engaged issue to a new/edited spec item vs. a gap (vs. dismissal).
- **Severity/triage** — whether issues carry severity and how triage prioritizes them.

Scope note: extends the SDD pipeline (`workflow` domain) with a new artifact type and
its reviewer/engage lifecycle. Exact decomposition settled during target-engage.
