# Target: Session screen opens target detail in a right-side panel, not a redirect

## Current statement

On the Session screen, clicking a target should reveal its detail **in a panel that
expands from the right side**, without leaving the Session screen.

1. **No redirect.** Today clicking a target row navigates away to the Targets screen
   (`onNav('targets', t.id)` in `hub/client/src/screens/Session.tsx`). Instead, the
   Session screen stays put and a detail panel slides/expands in from the right.

2. **Right-side expanding panel.** The target's detail view appears in a panel anchored to
   the right edge, so the session context remains visible alongside it. (Right-side
   placement is a required part of the request.)

3. **Shows the target detail.** The panel renders the target's full detail — naturally
   reusing the existing `TargetDetail` view (`hub/client/src/screens/Targets.tsx`) rather
   than building a second one.

4. **Dismissible.** The panel can be closed/collapsed to return to the plain session view.

Open decisions to settle at target-engage (not pinned here):
- **Reusable vs. session-specific** — whether the panel is a general right-side drawer
  component (usable later for gaps/work-items/spec rows) or specific to targets here.
- **Interaction details** — overlay vs. push-content, panel width, close affordances
  (close button / click-outside / Esc), and transition/animation.
- **Scope of rows** — the request is about targets; whether the other Session row types
  (gaps, work items, specs) adopt the same panel behavior or keep navigating is open.
- **URL / deep-link behavior** — whether an open panel is reflected in the route.

Scope note: primarily the Session screen (`ui-screens`); may introduce a reusable drawer
in `ui-components` if engage chooses the general approach. Exact decomposition settled
during target-engage.
