# Target: Syntax highlighting for code blocks in rendered markdown

## Current statement

The shared `Markdown` component (SPEC-uic-013) renders fenced code blocks with **syntax
highlighting** — language-aware coloring based on the code fence's language tag — instead of
flat monospaced text. The highlighting respects the app's existing token-based theming and the
light/dark theme.

1. Fenced code blocks with a language (e.g. ```` ```ts ````) render with highlighted tokens.
2. Plain fences with no language degrade gracefully (no errors, just unhighlighted code).
3. Highlight colors fit the existing theme (tokens / dark mode), not a clashing default scheme.

Open decisions to settle at target-engage (not pinned here):
- **Which highlighter** — `rehype-highlight` (highlight.js), Shiki, or `react-syntax-highlighter`
  — weighing bundle size, theming control, and SSR/none.
- **Theme integration** — map the highlighter's palette to the app's CSS tokens / dark mode.
- **Language coverage** — a common set vs. broad; and whether to lazy-load grammars.
- Keep raw HTML disabled (consistent with SPEC-uic-013); highlighting must not require it.

Scope note: extends the shared `Markdown` component (`ui-components`, builds on SPEC-uic-013/014).
Exact decomposition settled during target-engage.
