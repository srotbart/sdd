# Target: session-start primes the agent with the SDD model and essential context

## Current statement

`session-start` today renders a user-facing status snapshot. It should also **orient the
agent**: explain, at a high level, the shapes of the SDD artifacts, what each means, and
how to work with them — and load the essential context the agent needs to work well with
`.sdd/` for the rest of the session.

1. **Artifact orientation.** A concise, high-level explanation of each artifact type —
   targets, specs, gaps, work-items (and the newer issues / improvements / standards
   artifacts as they exist): what it represents, its lifecycle/state at a glance, and how
   the agent is expected to act on it. Not the full schema dump — the working mental model.

2. **The pipeline mental model.** The spec → code → review → gaps → refactor flow (the
   "TDD for agents" framing), so the agent understands where any given artifact sits and
   what the next move is, rather than treating each artifact in isolation.

3. **Essential context loading.** Surface the project-specific things the agent needs to
   work with this repo's `.sdd/` effectively — e.g. ID conventions, where artifacts live,
   active domains — so the agent starts informed instead of rediscovering it each session.

4. **Agent-facing, distinct from the user snapshot.** This orientation is for the agent's
   benefit; it complements (does not replace) the existing user-facing status report and
   the user-invoked `sdd-help` explanation.

Open decisions to settle at target-engage (not pinned here):
- **What counts as "essential context"** — artifact semantics, the pipeline/state machine,
  the agent-TDD framing, ID conventions, active domains; and how deep vs. linking out to
  `references/schemas.md`.
- **Delivery & verbosity** — always emitted vs. only when artifacts/skills are new or
  changed; how to keep it from bloating every session; concise summary vs. on-demand.
- **Relationship to `sdd-help`** — `sdd-help` is a user-invoked workflow explanation; this
  is agent-context priming at session start. Keep one source of truth for the model and
  avoid divergent copies.
- **Adaptivity** — whether the orientation reflects the project's actual artifact set
  (mention issues/improvements/standards only once those exist).

Scope note: extends the `session-start` skill (`workflow` domain) with agent-facing
orientation + context loading. Exact decomposition settled during target-engage.
